# Identity Architecture

家族・金融・連絡を「後から詰まない」ように設計するための思考メモ。

このリポジトリは、生活インフラを  
**設計・運用・スケーリングの観点から整理するドキュメント群**である。

具体的な契約内容や個別サービスではなく、  
**判断軸・原則・再利用可能な構造**のみを扱う。

---

## 構成（Table of Contents）

- Principles  
  └ 設計全体に共通する基本原則  
  → principles.md  

- Banking  
  └ 銀行（金融）が特異点になる理由  
  → banking.md  

- Numbering  
  └ 電話番号の役割分離とライフサイクル設計  
  → numbering.md  

- Devices  
  └ 端末の役割分離と事故半径の制御  
  → devices.md  

- Scaling  
  └ 人数・年齢・管理対象が増えた場合の拡張指針  
  → scaling.md  

---

## 全体構造（概念図）

```
人（人格）
└─ 中枢番号（固定・不可逆）
   ├─ 中枢端末（金融・本人確認）
   └─ 連絡用番号（副系統・可逆）
      └─ 連絡端末（Watch 等）

※ 人数が増えた場合は、この単位を複製する
```

---

## このリポジトリが扱わないもの

- 実際の契約内容・金額
- 利用中の銀行・通信事業者名
- 個人・家族が特定される情報
- 即効性のあるライフハック

---

## 想定読者

- 設計思考・構造化が好きな人
- 生活インフラを長期運用したい人
- 後から「なぜそう決めたか」を説明したい人

正解を提示するものではなく、  
**判断を再現可能にするための記録**である。
